## Highlight

- `roff tojson` now accepts `--indent N` to control JSON pretty-print indent width; default output remains compact.
- Release pipeline default BUILD_MODE switched from `native-first` to `zigbuild-first`, producing statically-linked binaries consistently across glibc/musl/macos without per-target toolchain installs.

## Add

- `roff tojson --indent N`: pretty-print JSON with N spaces of indent (accepts `--indent=N` form).
- `BUILD_MODE=zigbuild-first` default in `.x-cmd/release-rust.common.sh`; pass `--native-first` or `BUILD_MODE=native-first` to override.

## Change

- Removed `--pretty` flag (replaced by `--indent 2`).

## Internal

- Added `serde = "1"` dependency for `Serialize` trait used by custom JSON formatter.

## Note

- `cargo-zigbuild` must be installed for the default release flow; falls back to `cargo build` when unavailable.
